### 第3分钟：一键修复

#### 画面 7: 运行自动修复
```bash
$ agentguard fix openclaw

Fixing OpenClaw...
```

#### 画面 8: 修复过程（逐步显示）
```
[1/6] 🔴 端口暴露到所有网络接口
      修复方案: 将 bind 从 "lan" 改为 "loopback"
      影响: OpenClaw 将只能从本机访问
      是否应用? (y/n): y
      ✅ 已修复

      📝 备份已保存: ~/.openclaw/openclaw.json.backup.2026-03-08...

[2/6] 🔴 未启用身份认证
      修复方案: 启用 token 认证
      生成的 token: sk_ag_8x7k2m9n4p5q6r7s...
      已保存到: ~/.agentguard/tokens.txt
      是否应用? (y/n): y
      ✅ 已修复

[3/6] 🔴 DM 策略过于开放
      修复方案: 将 dmPolicy 从 "open" 改为 "pairing"
      影响: 新消息需要配对码验证
      是否应用? (y/n): y
      ✅ 已修复

[4/6] 🟡 未启用沙箱
      修复方案: 启用 Docker 沙箱
      是否应用? (y/n): y
      ✅ 已修复

[5/6] 🟡 工作空间权限为读写
      修复方案: 改为只读 (ro)
      是否应用? (y/n): y
      ✅ 已修复

[6/6] 🟡 配置文件权限过于宽松
      修复方案: chmod 600
      ✅ 已修复

──────────────────────────────────────────────────────
修复完成! OpenClaw 需要重启以应用更改.
是否立即重启? (y/n): y

● 正在重启 OpenClaw...
✅ OpenClaw 已成功重启

🎉 安全评分从 45 提升到 90!
```
