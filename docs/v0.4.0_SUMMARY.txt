🎯 优先级排序 (P0 必须完成)

  🥇 P0.1 桌面悬浮小部件 ⭐⭐⭐ (最高优先级)
     ├─ 悬浮球 (60x60 圆形, 状态颜色)
     ├─ 快速面板 (320x400, Agent状态+成本)
     └─ 系统托盘 (菜单栏图标+右键菜单)
     开发时间: 3-4 天

  🥈 P0.2 基础 Web Dashboard ⭐⭐⭐
     ├─ 总览页 (Agent列表+成本+评分)
     └─ Token统计页 (折线图+导出CSV)
     开发时间: 4-5 天

  🥉 P0.3 实时刷新 + WebSocket ⭐⭐
     ├─ 10秒定时刷新
     ├─ WebSocket推送
     └─ 手动刷新按钮
     开发时间: 1-2 天

  🏅 P0.4 预算告警系统 ⭐⭐⭐
     ├─ 预算配置 (日/周/月)
     ├─ 实时监控
     ├─ Dashboard横幅告警
     ├─ 桌面通知
     └─ 悬浮球颜色变化
     开发时间: 2-3 天

  📊 P0 总计: 10-14 天 (2 周)

🏆 为什么桌面模式是最高优先级？

  1. 差异化优势 ⭐⭐⭐
     → 其他工具没有桌面小部件

  2. 用户体验最佳 ⭐⭐⭐
     → 一键访问，无需打开浏览器
     → 常驻桌面，实时可见

  3. 可独立使用 ⭐⭐
     → 即使 Web Dashboard 未完成
     → 悬浮球已经很有价值

  4. 符合用户习惯 ⭐⭐⭐
     → 参考截图工具的交互方式
