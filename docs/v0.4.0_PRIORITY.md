## 🎯 用户需求：3 种使用方式

### 1️⃣ CLI 模式 - 终端运行
**已实现** ✅ (v0.3.0)
```bash
agentguard scan
agentguard tokens
agentguard export
```
**用户价值**: 快速检查，适合开发者

---

### 2️⃣ Web Dashboard 模式 - 浏览器运行
**v0.4.0 新增** 🆕
```bash

# → 浏览器打开 http://localhost:3000
```
**用户价值**: 实时监控，适合日常使用

---

### 3️⃣ 桌面小部件模式 - 悬浮窗口
**v0.4.0 新增** 🆕

**参考设计**: 类似截图工具的悬浮按钮
```
桌面悬浮球 → 点击展开 → 显示关键信息
├── 快速视图
│   ├── Agent 状态 (3/3 运行中)
│   ├── 今日成本 ($147)
│   └── 安全评分 (92/100)
│
└── 操作按钮
    ├── 打开 Dashboard
    ├── 快速扫描
    └── 设置
```

**用户价值**:
- 🎯 **一键访问** - 桌面直接点击
- 👀 **快速查看** - 不打开浏览器就能看关键数据
- 🔔 **实时告警** - 悬浮窗闪烁提示
- 📌 **常驻桌面** - 随时可用

## 🎨 桌面小部件详细设计

### 悬浮球设计

**尺寸**: 60x60 像素圆形

**状态显示**:
```
🟢 正常 (绿色)
   - 所有 Agent 运行正常
   - 成本在预算内
   - 无安全问题

🟡 警告 (黄色)
   - 预算使用 70%-90%
   - 有中低危安全问题

🔴 严重 (红色)
   - 预算超过 90% 或用尽
   - 有高危安全问题
   - Agent 停止运行

⚪ 离线 (灰色)
   - 无法获取数据
```

**交互**:
- 单击: 展开快速面板
- 右键: 系统托盘菜单
- 拖动: 移动位置
- 悬停: 显示提示 (Agent 数量 + 今日成本)

## 💡 开发策略

### 1. 先做桌面小部件 (差异化优势)

**原因**:
- ✅ 这是其他 AI 监控工具没有的
- ✅ 用户体验最好
- ✅ 可以独立使用 (不依赖 Web Dashboard)

**策略**:
- 悬浮球 + 快速面板先做出来

- ✅ 避免过度开发
- ✅ 快速验证用户需求
- ✅ 后续迭代优化

**MVP 范围**:

- Token 统计页 (必须)
- Agent 详情页 (可选 → P2)
- 设置页 (可选 → P2)

---

### 3. 复用现有代码

**原因**:
- ✅ 节省时间
- ✅ 已经测试验证
- ✅ 减少 Bug

**复用模块**:
- `src/core/*` - 100% 复用
- `src/i18n/*` - 100% 复用
