# AgentGuard 产品架构 - 用户视角

**从用户的角度看 AgentGuard 是如何工作的**

---

## 🎯 产品定位

**AgentGuard** = AI Agent 的"体检中心" + "账单管家"

- 📊 **监控**: 谁在运行？运行状态如何？
- 💰 **计费**: 花了多少钱？哪个最贵？
- 🛡️ **安全**: 有什么风险？如何修复？

## 🔧 用户可以操作的地方

### 1️⃣ 命令行操作

**位置**: 任何终端窗口

**可以做什么**:

```bash
# 扫描 AI Agents
$ agentguard scan
→ 看到: 检测到 3 个 Agent，安全评分 92/100

# 查看 Token 消耗
$ agentguard tokens
→ 看到: 今日 $147，本月 $1,792

# 生成 HTML 报告
$ agentguard export --format html
→ 生成: agentguard-report-2026-03-08.html
```

**特点**:
- ✅ 快速 (几秒钟)
- ✅ 简单 (一行命令)
- ❌ 需要重新运行才能看到新数据

---

### 2️⃣ HTML 报告查看

**位置**: 浏览器 (Chrome/Safari/Firefox)

**可以做什么**:
- 📊 查看安全评分
- 👀 查看 Agent 详情
- 💰 查看成本统计
- 🖨️ 打印或保存 PDF
- 📧 分享给团队 (发送 HTML 文件)

**特点**:
- ✅ 美观 (紫色渐变主题)
- ✅ 完整 (所有数据都在)
- ❌ 静态 (不会自动更新)
- ❌ 不能交互 (不能点击查看更多)

## 🎨 数据流向 (用户视角)

### 场景 1: 查看今天花了多少钱

```
1. 用户输入命令
   $ agentguard tokens

2. AgentGuard 读取数据
   → 读取 ~/.claude/projects/ 下的对话记录
   → 读取 ~/.cursor/ai-tracking/ 的数据库
   → 计算 Token 数量

3. 显示结果
   今日: $147.01
   - Claude Code: $141.84
   - Cursor IDE: $5.17
```

### 场景 2: 检查 Agent 是否安全

```
1. 用户输入命令
   $ agentguard scan

2. AgentGuard 检测
   → 查找系统中的 AI Agent 进程
   → 检查 Node.js 版本
   → 检查网络连接
   → 评估安全风险

3. 显示结果
   检测到 3 个 Agent
   总体评分: 92/100
   - Claude Code: 100/100 ✅
   - Cursor IDE: 100/100 ✅
   - OpenClaw: 75/100 ⚠️ (Node.js 版本过低)
```

### 场景 3: 生成月度报告

```
1. 用户输入命令
   $ agentguard export --format html

2. AgentGuard 生成
   → 收集所有数据
   → 生成 HTML 页面
   → 添加图表和样式

3. 输出文件
   agentguard-report-2026-03-08.html

4. 用户打开查看
   → 双击打开浏览器
   → 查看完整报告
   → 可以打印或分享
```

## 📊 数据来源和去向

### AgentGuard 读取的数据

```
📂 ~/.claude/
├── projects/
│   └── [project-id]/
│       └── transcripts/
│           └── *.jsonl  ← 读取对话记录，计算 Token
│
└── tasks/
    └── [task-id]/      ← 读取任务记录

📂 ~/.cursor/
└── ai-tracking/
    └── ai-code-tracking.db  ← 读取 SQLite 数据库

💻 系统进程
   $ ps aux | grep -E "claude|cursor|openclaw"
   ← 检测运行状态和 PID
```

### AgentGuard 生成的数据

```
📂 ~/.agentguard/
├── cache/
│   └── token-usage.db      ← SQLite 缓存历史数据
│
├── config.json             ← 用户设置 (预算、语言等)
│
└── logs/
    └── agentguard.log      ← 运行日志

📄 agentguard-report-*.html  ← HTML 报告文件
```

**用户不需要关心这些文件**，AgentGuard 自动管理

## 💡 关键设计原则

### 1. 简单启动

**v0.3.0** (当前):
```bash
agentguard scan       # 每次都要手动运行
agentguard tokens     # 每次都要手动运行
agentguard export     # 每次都要手动生成
```

**v0.4.0** (未来):
```bash
agentguard serve      # 一次启动，持续监控
```

---

### 2. 实时更新

**v0.3.0**: 静态数据
- ❌ 数据过时了需要重新生成

**v0.4.0**: 实时刷新
- ✅ 每 10 秒自动更新
- ✅ 数据变化立即推送
- ✅ 无需刷新页面

---

### 3. 交互探索

**v0.3.0**: 只能看
- ❌ 不能点击
- ❌ 不能缩放
- ❌ 不能筛选

**v0.4.0**: 可以交互
- ✅ 点击查看详情
- ✅ 缩放图表
- ✅ 筛选数据
- ✅ 导出多种格式

---

### 4. 主动告警

**v0.3.0**: 被动查看
- ❌ 用户要主动运行才知道
- ❌ 成本超了也不知道

**v0.4.0**: 主动通知
- ✅ 成本超标立即告警
- ✅ 桌面通知推送
- ✅ 邮件提醒 (可选)
