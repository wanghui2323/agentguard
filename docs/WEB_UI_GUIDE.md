# 🌐 AgentGuard Web UI 使用指南

## 概述

AgentGuard Web UI 提供了一个直观的可视化界面，让你无需使用命令行就能管理和监控 AI Agents 的安全状态。

## 快速开始

### 1. 启动 Web 服务器

```bash
cd agentguard
npm run web
```

你会看到：
```
╔═══════════════════════════════════════════════╗
║                                               ║
║   🛡️  AgentGuard Web UI 已启动               ║
║                                               ║
║   访问: http://localhost:3000                 ║
║                                               ║
╚═══════════════════════════════════════════════╝
```

### 2. 打开浏览器

访问 http://localhost:3000

### 3. 开始使用

点击 **"扫描所有 Agents"** 按钮，Web UI 会自动检测你电脑上运行的所有 AI Agents。

## 界面功能

### 主要操作按钮

#### 🔍 扫描所有 Agents
- **功能**：自动检测系统中运行的所有 AI Agents
- **扫描内容**：
  - Agent 运行状态
  - 端口配置
  - 安全漏洞
  - 配置问题

#### 🔧 一键修复全部
- **功能**：自动修复所有可以修复的安全问题
- **修复内容**：
  - 端口绑定配置
  - 身份认证设置
  - 权限调整
  - 配置文件权限
- **安全保证**：修复前自动备份，失败可回滚

#### 📄 导出报告
- **功能**：生成 JSON 格式的安全扫描报告
- **报告内容**：
  - 扫描时间戳
  - 所有 Agents 详细信息
  - 问题列表
  - 汇总统计

---

### 统计卡片

扫描完成后，页面顶部会显示 4 个统计卡片：

1. **检测到的 Agents**
   - 显示发现的 AI Agent 总数

2. **平均安全评分**
   - 所有 Agents 的平均安全分数（0-100）

3. **发现的问题**
   - 检测到的安全问题总数

4. **可自动修复**
   - 可以自动修复的问题数量

---

### Agent 卡片

每个检测到的 Agent 都会显示一个详细卡片：

#### 卡片头部
- **Agent 名称**：如 "OpenClaw"
- **安全评分**：0-100 分，带颜色编码：
  - ✅ 90-100: 优秀（绿色）
  - 🟢 70-89: 良好（浅绿）
  - 🟡 50-69: 需改进（黄色）
  - 🟠 30-49: 有风险（橙色）
  - 🔴 0-29: 危险（红色）

#### 状态信息
- **运行状态**：● 运行中 / ○ 已停止
- **端口信息**：监听的网络端口
- **进程 ID**：系统 PID

#### 问题列表
显示所有检测到的安全问题：
- **严重程度图标**：
  - 🔴 严重
  - 🟠 高危
  - 🟡 中危
  - 🟢 低危
  - ℹ️ 信息
- **问题标题**：简短描述
- **问题说明**：详细解释
- **修复标识**：✓ 可自动修复

#### 操作按钮
- **🔧 修复问题**：自动修复此 Agent 的所有可修复问题
- **⏸️ 停止**：停止 Agent 运行
- **🔄 重启**：重启 Agent
- **▶️ 启动**：启动已停止的 Agent

## 使用场景

### 场景 1: 首次安全检查

**步骤**：
1. 启动 Web UI
2. 点击 "扫描所有 Agents"
3. 查看每个 Agent 的安全评分
4. 阅读问题列表，了解安全风险
5. 决定是否需要修复

**示例输出**：
```
检测到 1 个 AI Agents
平均安全评分: 45/100

[OpenClaw] 评分: 45/100 🟠
发现 6 个问题 (3严重 2高危 1中危)
🔴 端口暴露到所有网络接口
🔴 未启用身份认证
...
```

---

### 场景 2: 一键修复安全问题

**步骤**：
1. 扫描完成后
2. 点击某个 Agent 卡片上的 "🔧 修复问题"
3. 等待修复完成（通常 3-5 秒）
4. 查看更新后的评分

**修复前后对比**：
```
修复前：45/100 🟠
修复后：90/100 🟢

修复内容：
✅ 端口改为本地访问
✅ 启用身份认证（自动生成强令牌）
✅ DM 策略设为配对模式
✅ 启用沙箱隔离
✅ 工作空间改为只读
```

---

### 场景 3: 管理 Agent 生命周期

**停止 Agent**：
```
点击 "⏸️ 停止" → Agent 停止运行 → 状态变为 "○ 已停止"
```

**重启 Agent**：
```
点击 "🔄 重启" → Agent 重新启动 → 应用最新配置
```

**使用场景**：
- 修复配置后需要重启生效
- 临时停止 Agent 节省资源
- 切换不同 Agent 配置

---

### 场景 4: 导出合规报告

**步骤**：
1. 扫描完成后
2. 点击 "📄 导出报告"
3. 浏览器自动下载 JSON 文件
4. 文件名格式：`agentguard-report-{timestamp}.json`

**报告用途**：
- 企业合规审计
- 安全团队汇报
- 历史记录存档
- 问题追踪

**报告示例**：
```json
{
  "timestamp": "2026-03-08T10:30:00.000Z",
  "summary": {
    "total": 1,
    "avgScore": 45,
    "totalIssues": 6
  },
  "agents": [
    {
      "id": "openclaw",
      "name": "OpenClaw",
      "score": 45,
      "issues": [...]
    }
  ]
}
```

## 技术架构

### 前端
- **纯 HTML + JavaScript**：无需额外构建工具
- **响应式设计**：支持桌面和平板
- **现代 UI**：渐变色、卡片布局、动画效果

### 后端
- **Node.js HTTP 服务器**：轻量级，零依赖
- **RESTful API**：标准化接口
- **集成 CLI 核心**：复用扫描、修复、控制逻辑

### API 端点

| 端点 | 方法 | 功能 |
|------|------|------|
| `/api/scan` | GET | 扫描所有 Agents |
| `/api/fix/:agentId` | POST | 修复指定 Agent |
| `/api/stop/:agentId` | POST | 停止指定 Agent |
| `/api/restart/:agentId` | POST | 重启指定 Agent |
| `/api/status` | GET | 获取所有 Agent 状态 |

## 高级用法

### 自定义端口

如果 3000 端口被占用，可以修改 `src/server/index.ts`：

```typescript
const PORT = 3000; // 改为其他端口，如 8080
```

### 远程访问

默认只能本机访问（`localhost`）。如需远程访问：

**⚠️ 警告**：远程访问有安全风险，建议仅在安全网络中使用

1. 修改 `src/server/index.ts`：
```typescript
server.listen(PORT, '0.0.0.0', () => { // 监听所有网络接口
  console.log(`访问: http://0.0.0.0:${PORT}`);
});
```

2. 建议添加身份验证（未来版本）

### 自动刷新

可以添加定时自动扫描功能：

```javascript
// 在 Web UI 中添加
setInterval(async () => {
  await scanAgents();
}, 60000); // 每分钟自动扫描
```

## 常见问题

### Q: 扫描失败，显示 "请确保已启动 AgentGuard 服务器"？

**A**: 检查是否启动了 Web 服务器：
```bash
npm run web
```

### Q: 修复后评分没有提升？

**A**: 可能的原因：
1. 问题无法自动修复（如 Node.js 版本）
2. 修复失败，查看控制台日志
3. 需要重启 Agent 后生效

### Q: 能否同时打开多个浏览器标签页？

**A**: 可以，但操作会互相影响。建议：
- 一个标签页用于监控
- 另一个标签页用于操作

### Q: Web UI 和 CLI 能同时使用吗？

**A**: 可以，它们共享相同的后端逻辑。但注意：
- CLI 修复后，Web UI 需要手动刷新
- 建议主要使用一种方式，避免混淆

## 性能优化

### 减少扫描时间
- 扫描速度取决于 Agent 数量
- 单个 Agent 扫描约 1-2 秒
- 3 个 Agent 预计 3-6 秒

### 降低内存占用
- Web UI 内存占用约 20-30MB
- 服务器内存占用约 50-80MB
- 总计约 100MB

---

## 安全建议

1. **仅本地访问**：不要将 Web UI 暴露到公网
2. **定期扫描**：每天至少扫描一次
3. **及时修复**：评分低于 70 应立即修复
4. **备份配置**：修复前会自动备份，但建议手动备份重要配置
